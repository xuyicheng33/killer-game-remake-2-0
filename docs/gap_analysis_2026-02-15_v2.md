## 当前实现状态矩阵

| 模块 | 原版要求（最低） | 当前状态 | 缺口结论 |
|---|---|---|---|
| `run_meta` | 一局流程状态（楼层/金币/血量/章节） | 已有最小 `RunState` | 缺章节推进、难度、跨战斗奖励写回 |
| `map_event` | 多层分支地图 + 事件节点 | 仅单层候选节点 | 缺真实路径图、节点权重、事件脚本 |
| `battle_loop` | 明确阶段状态机 | 可玩，但仍为legacy战斗编排 | 缺可测试的阶段机与统一行动窗口 |
| `card_system` | 四牌区 + 目标 + 关键词机制 | 手牌拖拽与出牌可用 | 缺消耗堆可视化、保留/消耗/X费等关键词框架 |
| `effect_engine` | 顺序结算栈 | 直接即时生效 | 缺效果队列、触发优先级、可回放日志 |
| `buff_system` | 力量/敏捷/易伤/虚弱/中毒等 | 基本无独立系统 | 关键缺口，必须优先补 |
| `enemy_intent` | 意图+规则约束+进阶数值 | 条件+权重基础可用 | 缺不可连续动作约束和进阶参数化 |
| `reward_economy` | 战后奖励/商店/删卡/金币 | 仅金币占位 | 缺奖励页、商店全流程 |
| `relic_potion` | 遗物触发链/药水槽 | 未实现 | 全量缺失 |
| `seed_replay` | 存档、固定种子复现 | 仅开局seed占位 | 缺保存、读档、战斗复现能力 |
| `content_pipeline` | 数据导入校验、版本迁移 | 仅参考索引脚本 | 缺结构化数据源与校验工具 |
| `ui_shell` | 1080p可读、中文UI、可替换皮肤 | 基础可见 | 缺完整样式系统与全界面视觉统一 |

## 结论（必须先补）

1. 第一优先：`effect_engine + buff_system`，否则卡牌和遗物规模上不去。
2. 第二优先：`battle_loop` 状态机化，减少后续回归成本。
3. 第三优先：`reward_economy + map_event`，形成可连续推进的一局。
4. 资源侧：当前美术/音乐仍为教程基线，需独立任务整包替换。
