 7-3：出牌后卡牌卡屏修复

 优先级：P1（严重体验 Bug）
 关键文件：

 现象：拖动卡牌打出后，卡牌节点未被移除/隐藏，卡在屏幕中央，下回合才消失

 - 添加防重入保护，避免同一张卡被多次处理

 7-4：击杀所有敌人立即判定胜利

 任务ID：fix-enemy-kill-immediate-victory-v1
 优先级：P1（核心流程 Bug，关联 P1-2 高风险竞态）
 关键文件：
 - runtime/modules/buff_system/buff_system.gd（_handle_death 方法，第 334-346 行）
 - runtime/modules/battle_loop/battle_phase_state_machine.gd（check_battle_end）
 - runtime/modules/battle_loop/battle_context.gd
 - runtime/global/events.gd

 现象：击杀最后一个敌人后，战斗不立即结束，需手动结束回合才触发胜利

 根因（P1-2）：
 BuffSystem._handle_death() 直接调用 target.queue_free()，但胜负判定在 RESOLVE_PHASE 末尾。
 敌人死亡信号（Events.enemy_died）在 ACTION_PHASE 内发出，但状态机仍等待 RESOLVE_PHASE 才检查。

 修复方案：
 1. _handle_death() 不直接 queue_free()，改为发送 Events.entity_death_requested.emit(target)
 2. battle_phase_state_machine.gd 监听 entity_death_requested，在当前 phase 末尾统一处理死亡
 3. 死亡后检查 check_battle_end()，若所有敌人已死则立即触发胜利流程
 4. 节点的 queue_free() 统一在死亡处理队列里按序执行

 GUT 测试：
 - test_killing_all_enemies_triggers_immediate_victory()
 - test_dot_death_triggers_battle_end_correctly()

 验收：击杀最后一个敌人后（含 DOT 击杀），战斗立即结束，无需手动结束回合

 7-5：遗物触发对象缓存（P1-4 性能修复）

 任务ID：fix-relic-runtime-cache-v1
 优先级：P1（性能 + 状态追踪问题）
 关键文件：runtime/modules/relic_potion/relic_potion_system.gd（第 185-194 行）

 现象：每次遗物触发（出牌、受伤、回合开始等）都实例化全新遗物对象，导致：
 1. 无法维护跨触发的遗物状态（如"本战已触发X次"）
 2. 频繁 GC 压力

 修复方案：
 1. RelicPotionSystem 初始化时为每个遗物创建单一运行时对象，存入字典 _relic_instances: Dictionary
 2. 触发时从缓存中取对象，不再每次 create_relic()
 3. 战斗结束时根据遗物类型决定是否重置状态（跨战斗遗物保持，单战遗物清空）

 验收：
 - 有跨触发状态的遗物（如"第3次受伤时获得力量"）行为正确
 - GUT 测试：test_relic_runtime_object_is_reused_across_triggers()
