# 基础版执行计划（可逐任务派发）

目标：先做一个稳定可玩的基础版，再做扩展。
原则：每个任务完成后你都能直接运行对比。

## T00（已完成）`feat-bootstrap-v0-20260215`
- 模块：`app + run_meta + map_event`
- 目标：建立根目录可运行骨架（地图->战斗->结算）。
- 预期结果：启动进入地图，点节点进战斗，胜负后回流程。
- 运行验收：Godot 打开根项目，运行 `res://runtime/scenes/app/app.tscn`。

## T01 `feat-battle-state-machine-v1`
- 模块：`battle_loop`
- 目标：把战斗阶段抽为明确状态机（抽牌阶段/行动阶段/敌方阶段/结算阶段）。
- 预期结果：战斗右上角显示当前阶段文本，阶段切换可见。
- 运行验收：打一回合，观察阶段文字按顺序变化。

## T02 `feat-card-zones-v1`
- 模块：`card_system`
- 目标：完善牌区（抽牌堆/弃牌堆/消耗堆）并可视化数量。
- 预期结果：战斗 UI 新增三个计数器，出牌与回合结束时数字变化正确。
- 运行验收：打出会消耗的测试牌，看到“消耗堆”计数增加。

## T03 `feat-effect-stack-v1`
- 模块：`effect_engine`
- 目标：引入效果队列（顺序结算），替换直接即时生效。
- 预期结果：高亮显示“待结算效果数”，多段伤害逐个结算。
- 运行验收：使用多段攻击卡，观察逐段跳字而非一次合并。

## T04 `feat-buff-system-v1`
- 模块：`buff_system`
- 目标：实现基础状态：力量、敏捷、虚弱、易伤、中毒。
- 预期结果：单位头像下出现状态图标与层数。
- 运行验收：用施加虚弱/中毒的卡，观察状态生效与回合衰减。

## T05 `feat-enemy-intent-rules-v1`
- 模块：`enemy_intent`
- 目标：敌人意图改为“规则+权重”模型，支持不连续动作约束。
- 预期结果：同一敌人不会出现非法连续动作，意图展示稳定。
- 运行验收：重复进入战斗数次，检查敌人行为序列是否符合规则。

## T06 `feat-reward-flow-v1`
- 模块：`reward_economy`
- 目标：战后奖励界面（金币 + 三选一卡）。
- 预期结果：胜利后不直接回地图，先进入奖励页，选完再回地图。
- 运行验收：打赢一场，确认奖励页出现并可选卡。

## T07 `feat-map-progression-v1`
- 模块：`map_event`
- 目标：实现基础多层地图与路径推进。
- 预期结果：地图不再是单层候选，出现层级推进与已走路径标识。
- 运行验收：连续通过 3 层，确认层数递增且节点状态更新。

## T08 `feat-rest-event-v1`
- 模块：`map_event`
- 目标：实现营火（休息/升级）与 3 个高频事件。
- 预期结果：进入休息点可选项生效；事件弹窗含分支选择。
- 运行验收：走到休息点恢复生命，再触发事件并看到分支结果。

## T09 `feat-relic-potion-core-v1`
- 模块：`relic_potion`
- 目标：接入遗物栏与药水栏，先做 10 遗物 + 8 药水。
- 预期结果：局内可获得并显示，部分触发时机生效（战斗开始/出牌后/受击后）。
- 运行验收：拿到样例遗物后进战斗，观察触发提示。

## T10 `feat-shop-v1`
- 模块：`reward_economy`
- 目标：实现商店（买卡/买遗物/删卡）。
- 预期结果：商店节点可进入，金币正确扣减，购买结果持久化到 Run。
- 运行验收：进商店买一张卡，再进战斗确认已加入牌组。

## T11 `feat-save-seed-v1`
- 模块：`seed_replay`
- 目标：实现存档/读档与固定种子开局。
- 预期结果：主菜单可“继续游戏”；相同种子生成一致地图与首战敌人。
- 运行验收：保存后重开恢复进度；输入同种子两次开局对比结果一致。

## T12 `feat-ui-revamp-v1`
- 模块：`ui_shell`
- 目标：UI 视觉大幅优化（中文字体体系、卡牌重绘、HUD 重构、地图重绘）。
- 预期结果：整体视觉明显区别于教程基线，同时保留交互手感。
- 运行验收：对比截图（旧版/新版），检查桌面与 16:9 缩放兼容。

## T13 `feat-act1-content-pack-v1`
- 模块：`content_pipeline`
- 目标：基础版内容包：1 角色、30 卡、12 敌组、30 遗物、10 事件。
- 预期结果：可完成一幕流程（含 Boss）并结束结算。
- 运行验收：完整跑通一局 Act1（普通难度）。

## T14 `feat-regression-tests-v1`
- 模块：`battle_loop + effect_engine + seed_replay`
- 目标：加入关键回归测试（状态叠加、结算顺序、种子复现、存档兼容）。
- 预期结果：每次改动可快速验证核心机制不回退。
- 运行验收：执行测试命令全部通过。
